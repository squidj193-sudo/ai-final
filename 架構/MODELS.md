# 模型策略：AI 研究助理 Agent

本文件說明 AI 研究助理所使用的語言模型及其選擇理由。

## 1. 主要模型：Gemini 2.5 Flash

系統核心完全由 **Gemini 2.5 Flash** 驅動。

### 為何選擇 Gemini 2.5 Flash？
- **極速回應**：針對低延遲優化，在對話介面中提供近乎即時的回應。
- **大型上下文支援**：可同時處理多篇完整的研究論文，適合文獻矩陣 Skill 的需求。
- **高效成本**：相較於 Pro 模型，以更低成本提供高效能推理，適合大量 RAG 操作。
- **多模態能力**：為未來分析論文中的圖表做好技術儲備。

## 2. 各技能模組的模型應用

| 技能模組 | 模型用途 |
| :--- | :--- |
| **Agent Core** | 意圖分類與工具協調。 |
| **論文搜尋** | 從使用者查詢中提取語意關鍵字。 |
| **文獻分析** | 對 Markdown 格式的論文進行結構化摘要。 |
| **文獻矩陣** | 跨多篇論文摘要進行比較分析。 |
| **方向分析** | 識別研究缺口的策略性推理。 |

## 3. 設定參數
- **Temperature（溫度）**：
    - 分析與矩陣技能為 0.2（高一致性）。
    - 方向分析技能為 0.7（鼓勵具創意的研究建議）。
- **最大輸出 Token 數**：4096（統一標準）。
- **安全設定**：標準企業級安全過濾。

## 4. 效能監控
系統監控每個技能的 Token 用量與回應延遲，以確保 `gemini-2.5-flash` 模型持續滿足研究助理的效能需求。
