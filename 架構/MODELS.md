# 模型策略：AI 研究助理 Agent

本文件說明 AI 研究助理所使用的語言模型及其選擇理由。

## 1. 雙模型備援策略

為提供兼具「高效能對話推理」與「高可用性服務防護」的體驗，系統採用了雙模型備援策略。

### 主要模型：Gemini 3.5 Flash (`gemini-3.5-flash`)
- **優異的推理能力**：作為新一代輕量級大語言模型，提供極佳的學術理解與邏輯推理能力。
- **低延遲回應**：針對速度進行深度優化，在對話與意圖判定上提供流暢的高響應速度。
- **大型上下文支援**：適合多文獻 RAG 與對比矩陣生成的輸入吞吐。

### 備援模型：Gemma 4 26B (`gemma-4-26b-a4b-it`)
- **無感切換**：當主要模型觸發 429 流量配額限制 (ResourceExhausted) 時，後端 `FallbackGenerativeModel` 會自動捕獲異常並切換至備援模型。
- **優異的開源推理**：做為開源的高性能中量級模型，保障學術研究功能的連續性。

---

## 2. 各技能模組的模型應用

| 技能模組 | 模型用途 |
| :--- | :--- |
| **Agent Core** | 意圖分類與工具協調。 |
| **論文搜尋** | 從使用者查詢中提取語意關鍵字與自動英譯。 |
| **文獻分析** | 對 Markdown 格式的論文進行結構化摘要。 |
| **文獻矩陣** | 跨多篇論文摘要進行比較分析。 |
| **論文圖譜** | 生成代表性社群標籤。 |
| **方向分析** | 識別研究缺口的策略性推理。 |

---

## 3. 設定參數

- **Temperature（溫度）**：
    - 分析與矩陣技能為 0.2（高一致性）。
    - 方向分析與一般對話技能為 0.7（鼓勵具創意的研究建議）。
- **最大輸出 Token 數**：
    - 一般對話與意圖判定：1024 / 128
    - 分析、矩陣、與方向技能：4096
- **安全設定**：標準企業級安全過濾。

---

## 4. 效能監控

系統監控每個技能的 Token 用量與回應延遲，以確保 `gemini-3.5-flash` 與 `gemma-4-26b-a4b-it` 模型持續滿足研究助理的效能需求。
