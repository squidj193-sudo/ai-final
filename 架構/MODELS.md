# 模型策略：AI 研究助理 Agent

本文件說明 AI 研究助理所使用的語言模型及其選擇理由。

## 1. 主要模型：Gemma 4 26B

系統核心完全由 **Gemma 4 26B (gemma-4-26b-a4b-it)** 驅動。

### 為何選擇 Gemma 4 26B？
- **優異的推理能力**：作為新一代輕量級大語言模型，提供極佳的學術理解與邏輯推理能力。
- **高效回應**：針對低延遲進行優化，在對話介面中提供流暢的回應速度。
- **大型上下文支援**：可同時處理多篇完整的研究論文，適合文獻矩陣 Skill 的需求。
- **學術與開源友好**：適合各類研究助理與自訂 RAG 應用。

## 2. 各技能模組的模型應用

| 技能模組 | 模型用途 |
| :--- | :--- |
| **Agent Core** | 意圖分類與工具協調。 |
| **論文搜尋** | 從使用者查詢中提取語意關鍵字。 |
| **文獻分析** | 對 Markdown 格式的論文進行結構化摘要。 |
| **文獻矩陣** | 跨多篇論文摘要進行比較分析。 |
| **方向分析** | 識別研究缺口的策略性推理。 |

## 3. 設定參數
- **Temperature（溫度）**：
    - 分析與矩陣技能為 0.2（高一致性）。
    - 方向分析技能為 0.7（鼓勵具創意的研究建議）。
- **最大輸出 Token 數**：4096（統一標準）。
- **安全設定**：標準企業級安全過濾。

## 4. 效能監控
系統監控每個技能的 Token 用量與回應延遲，以確保 `gemma-4-26b-a4b-it` 模型持續滿足研究助理的效能需求。
