# 產品需求文件（PRD）：AI 研究助理 Agent

## 1. 專案概述
**AI 研究助理 Agent** 是一套智慧型系統，旨在精簡學術研究流程。它協助使用者透過關鍵字搜尋論文、上傳文獻進行分析、自動生成比較矩陣，並根據「角色狀態（Role State）」（大方向 / 中方向 / 小方向）提供可行的研究方向建議。

## 2. 目標使用者與痛點

### 目標使用者
- 博士生與碩士生。
- 進行專題研究的大學生。
- 正在探索新領域的研究人員。

### 主要痛點
1. **資訊過載**：難以追蹤數量龐大的學術論文。
2. **整理費力**：手動建立文獻比較矩陣費時且容易出錯。
3. **研究缺口難以發掘**：初學者難以從現有文獻中找出可行的研究問題。
4. **範疇管理困難**：難以從廣泛主題縮小到具體的研究問題。

## 3. 產品目標與成功指標
- **效率**：將初步文獻回顧所需時間至少縮短 50%。
- **準確性**：確保 AI 生成的摘要與比較表格忠實反映原始資料。
- **引導性**：提供符合使用者研究範疇的結構化研究方向建議。

## 4. 核心工作流程

### 流程 A：探索（關鍵字搜尋）
1. 使用者定義**角色狀態**（例：「光電」→「太陽能電池」→「鈣鈦礦」）。
2. 使用者輸入搜尋關鍵字。
3. Agent 呼叫 `search_papers` 工具，依角色狀態篩選結果。
4. Agent 回傳論文清單與摘要。

### 流程 B：深度分析（上傳文獻）
1. 使用者上傳一篇或多篇 PDF 論文。
2. Agent 呼叫 `parse_paper`（透過 MarkItDown）將 PDF 轉為 Markdown。
3. Agent 呼叫 `summarize_paper` 生成結構化摘要。
4. 使用者要求進行比較。
5. Agent 呼叫 `build_literature_matrix`，生成比較方法、發現與限制的 Markdown 表格。

### 流程 C：研究方向建議
1. Agent 分析比較矩陣與現有摘要。
2. Agent 識別「研究缺口」（如：缺乏實驗驗證、特定環境條件等）。
3. Agent 呼叫 `analyze_directions`，提出 3-5 個可行的研究主題。

## 5. 功能需求（技能模組）

| 技能模組 | 說明 | 主要職責 |
| :--- | :--- | :--- |
| **論文搜尋** | 學術搜尋整合 | 呼叫 API（Semantic Scholar、arXiv），依角色狀態篩選並回傳元資料。 |
| **文獻分析** | 文件處理 | PDF 轉 MD、結構化摘要、RAG 知識庫匯入。 |
| **文獻矩陣** | 比較合成 | 將多篇論文摘要彙整成結構化比較表格。 |
| **方向分析** | 策略引導 | 識別矩陣中的研究缺口並提出研究方向。 |
| **角色狀態管理** | 上下文持久化 | 維護使用者的階層式研究範疇（大 / 中 / 小方向）。 |

## 6. 技術架構
系統採用**多技能單一 Agent 架構**。

- **Agent Core**：由 LLM 驅動的協調器，負責解讀使用者意圖並選擇適當的技能或工具。
- **工具**：用於搜尋、解析與分析的原子函式。
- **RAG 系統**：
    - **解析**：MarkItDown 進行高保真度的 MD 轉換。
    - **儲存**：向量資料庫（如 ChromaDB）用於語意檢索論文片段。
    - **上下文**：將檢索到的片段注入 LLM Prompt，提供有依據的回答。

## 7. 介面設計規格
- **主對話介面**：左側固定顯示「對話列表」，中央為支援 PDF 上傳的對話區域。
- **摘要卡片**：顯示論文摘要的視覺元件，並提供快速操作（如「加入比較」）。
- **矩陣檢視**：專用的表格頁面，可比較多篇論文，支援匯出為 Markdown/CSV。
- **方向報告**：顯示研究建議的結構化報告頁面，以樹狀圖呈現領域層級結構。

## 8. 資料與隱私策略
- **來源標示**：摘要或矩陣中的每項主張均須連結至特定論文（標題、DOI、作者）。
- **格式**：所有內部資料交換使用 JSON；使用者端報告使用 Markdown。
