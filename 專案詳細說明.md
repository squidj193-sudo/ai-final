# 🔬 AI 研究助理 Agent 專案詳細說明文件

本專案是一套基於 **Gemma 4 26B (gemma-4-26b-a4b-it)** 模型所建構的學術研究輔助系統。本系統專為研究人員打造，提供從「學術文獻搜尋與過濾」、「PDF 論文上傳與 RAG 語意索引」、「結構化論文摘要提取」，到「多文獻比較矩陣生成」與「研究缺口及方向分析建議」的一站式工作流程。

---

## 🏗️ 系統架構圖 (Architecture Overview)

本系統採用前後端分離（Decoupled）架構：


## ⚙️ 後端核心技術模組與邏輯

### 1. 意圖檢測與分派 (`agent_core.py`)
`AgentCore` 是整套系統的「大腦」，負責接收使用者的訊息，並使用一個輕量級的 Prompt 詢問 Gemini 模型，將使用者的輸入分類為以下幾種意圖：
*   `search` (搜尋論文)
*   `analyze` (分析論文)
*   `matrix` (生成比較矩陣)
*   `direction` (分析研究方向)
*   `set_direction` (設定研究領域狀態)
*   `chat` (一般學術對話)

一旦識別出意圖後，`AgentCore` 就會調用對應的 `Skill` 模組，並將處理結果格式化為前端所需的結構資料。

### 2. 角色與研究方向狀態 (`skills/state_skill.py`)
系統支援維護一個層級式的研究方向狀態：
*   **大方向** (如：綠色能源)
*   **中方向** (如：太陽能技術)
*   **小方向** (如：鈣鈦礦太陽能電池元件效能)

當使用者在前端設定好研究方向後，該上下文會被自動加入到 **學術搜尋 (Semantic Scholar)** 的查詢字詞中，以及作為 **Gemini 生成提示詞** 的 Context，從而實現高度個人化的文獻推薦與建議。

### 3. 文獻搜尋與擷取 (`skills/search_skill.py`)
本模組直接串接 **Semantic Scholar API**。
當用戶執行搜尋時，系統會將使用者的「搜尋關鍵字」與「目前研究方向上下文」進行拼接，自動向 Semantic Scholar 獲取最相關的學術文獻，包括標題、作者、發表年份、摘要、URL 與 DOI。

### 4. RAG 與 PDF 處理機制 (`tools/rag.py`)
為了解析用戶上傳的本地 PDF 文獻，系統配置了輕量化 RAG (Retrieval-Augmented Generation) 機制：
1.  **MarkItDown 解析**：使用微軟的 `markitdown` 工具，將上傳的 PDF 論文精準轉化為純文字/Markdown。
2.  **文本切塊 (Chunking)**：使用 Sliding Window 演算法，將大篇幅論文切分為多個 Chunks（預設大小 1500 字，重疊 300 字）。
3.  **本地檢索與儲存**：將解析出的 Markdown 與 Metadata 保存至本地 `./data/papers` 目錄。檢索時，系統使用基於詞頻（TF-IDF 概念）的本機檢索演算法，比對使用者提問並回傳最相關的文本片段，無需繁重的外部向量資料庫。

### 5. 摘要與矩陣生成 (`skills/analysis_skill.py`, `skills/matrix_skill.py`)
*   **結構化摘要**：使用 Gemma 4 26B 分析論文前導內容，強迫模型以嚴格的 JSON Schema 輸出包含 `research_goal` (研究目的)、`methodology` (研究方法)、`main_findings` (主要發現)、`limitations` (研究限制) 與 `keywords` (關鍵字) 的結構化數據。
*   **比較矩陣**：當系統中累積至少 2 篇已分析的摘要時，`MatrixSkill` 會將這些結構化摘要彙整成 JSON 傳給 Gemini，並命令其渲染為 Markdown 格式的文獻比較表格，並在文末附上「研究缺口分析」。

## 🎨 前端功能與頁面設計

前端由 Vite + React 驅動，介面採現代深色極簡風（Dark Mode），提供高度互動的卡片式設計與動態載入動畫：

1.  **對話與搜尋主頁 (`ChatPage.jsx`)**
    *   提供一個聊天對話窗，可以直接輸入自然語言指令（如：「幫我搜尋鈣鈦礦太陽能的最新論文」）。
    *   點選搜尋按鈕會觸發 Semantic Scholar API，並在側邊以精美卡片渲染文獻列表。
    *   **PDF 上傳模組**：支援拖曳或選擇本地 PDF 檔案上傳。上傳後自動進入 RAG 解析與摘要分析流程。
2.  **文獻摘要記錄頁 (`SummaryPage.jsx`)**
    *   列出當前 Session 所有已被系統分析的文獻。
    *   每一篇文獻都以收合風琴式卡片（Accordion）呈現其「研究目的」、「研究方法」、「主要發現」和「研究限制」。
3.  **比較矩陣頁 (`MatrixPage.jsx`)**
    *   當前已儲存論文超過 2 篇時，點擊即可一鍵產生 Markdown 文獻比較表格。
    *   支援語意化的「研究缺口分析」區塊。
4.  **研究方向建議頁 (`DirectionPage.jsx`)**
    *   基於文獻矩陣的缺口分析，調用 `DirectionSkill` 生成 3-5 個極具可行性的未來研究課題，包含具體題目、方法、預期貢獻與可行性評估（高/中/低），為研究者提供靈感。

## 🚀 啟動與配置步驟

請確保本機已安裝 **Python 3.10+** 及 **Node.js 18+**。

### 1. 後端設定
1. 進入 `backend` 目錄：
   ```bash
   cd backend
   ```
2. 複製並設定 `.env` 環境變數：
   ```bash
   cp .env.example .env
   ```
   在 `.env` 中設定您的 `GEMINI_API_KEY`：
   ```env
   GEMINI_API_KEY="您的_Gemini_API_Key"
   GEMINI_MODEL="gemma-4-26b-a4b-it"
   ```
3. 建立並啟用虛擬環境（可選，專案中已包含 venv）：
   ```bash
   python -m venv venv
   # Windows 啟用:
   .\venv\Scripts\activate
   ```
4. 安裝依賴：
   ```bash
   pip install -r requirements.txt
   ```
5. 啟動後端伺服器：
   ```bash
   python main.py
   ```
   後端將執行於 `http://localhost:8000`。

### 2. 前端設定
1. 進入 `frontend` 目錄：
   ```bash
   cd frontend
   ```
2. 安裝依賴套件：
   ```bash
   npm install
   ```
3. 啟動前端開發伺服器：
   * *注意：若在 Windows PowerShell 下遇到執行權限限制 (PSSecurityException)，請使用下列指令啟動：*
   ```bash
   npm.cmd run dev
   ```
   前端將執行於 `http://localhost:5173`。
